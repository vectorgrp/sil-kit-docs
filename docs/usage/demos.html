Demos¶

This document describes the usage of the demo projects that are included with the Vector SIL Kit project and what their expected output and or results are. All demo source code is located in the Git repository in the folder Demos.

Note

All paths on the following pages are relative to the top level of the Git repository. Build artifacts are assumed to be located in a build subdirectory. For simplicity’s sake all paths assume a Release configuration.

To build the demos, please refer to Building the Demos.

Building the Demos¶

Building the demos from within the source tree is straight forward, just build the Demos CMake target:

cmake --build . --target Demos

The individual demos are build as a dependency.

Note

The distributed Demos, as packaged by CPack, are preconfigured to build against a copy of the SIL Kit binaries in ../SilKit/ . This can be overriden by providing your own SilKit CMake target library, before the demos are configured by cmake. Or by changing the find_package(SilKit ... PATHS path/to/SilKit) statement directly in the SilKit-Demos/CMakeLists.txt directory.

CAN Demo¶

Abstract CAN Reader/Writer with or without network simulator
Source location Demos/Can
Requirements
  • sil-kit-system-controller (not needed for unsynchronized execution)
  • sil-kit-system-monitor (optional)
  • network simulator (optional)
Parameters
<ParticipantConfiguration.json|yaml>
File name of the ParticipantConfiguration to be used; use SilKitConfig_DemoCan.json for an example configuration.
<ParticipantName>
The name of the participant within the simulation; must either be CanWriter or CanReader.
[RegistryUri]
The silkit:// URI of the registry to connect to; defaults to silkit://localhost:8500 (optional).
[–async]
If async flag is set, the participant will join the simulation unsynchronized and it will not need the build/Release/sil-kit-system-controller to start.
Parameter Example 
# Creates a CAN Writer Process with the default registry URI
build/Release/SilKitDemoCan Demos/Can/SilKitConfig_DemoCan.json CanWriter
System Example

For synchronized execution:

# System Monitor (optional):
build/Release/sil-kit-system-monitor

# CAN Reader:
build/Release/SilKitDemoCan Demos/Can/SilKitConfig_DemoCan.json CanReader

# CAN Writer:
build/Release/SilKitDemoCan Demos/Can/SilKitConfig_DemoCan.json CanWriter

# System Controller:
build/Release/sil-kit-system-controller CanReader CanWriter

For unsynchronized execution:

# CAN Reader:
build/Release/SilKitDemoCan Demos/Can/SilKitConfig_DemoCan.json CanReader --async

# CAN Writer:
build/Release/SilKitDemoCan Demos/Can/SilKitConfig_DemoCan.json CanWriter --async
Notes
- The writer sends CAN frames at a fixed rate of one frame per simulation step (1ms).
- Both reader and writer sleep for 1 second per quantum to slow down execution.

LIN Demo¶

Abstract LIN Master and Slave demo. The master sends and requests messages from a LIN slave.
Source location Demos/Lin
Requirements
  • sil-kit-system-controller
  • sil-kit-system-monitor (optional)
  • Network simulator (optional)
Parameters
<ParticipantConfiguration.json|yaml>
File name of the ParticipantConfiguraiton to be used; use SilKitConfig_DemoLin.json for an example configuration.
<ParticipantName>
The name of the participant within the simulation; must either be EthernetWriter or EthernetReader.
[RegistryUri]
The silkit:// URI of the registry to connect to; defaults to silkit://localhost:8500 (optional).
Parameter Example 
# Creates a LIN Master Process with the default registry URI:
build/Release/SilKitDemoLin Demos/Lin/SilKitConfig_DemoLin.json LinMaster
System Example 
# System Monitor (optional):
build/Release/sil-kit-system-monitor

# LIN Master:
build/Release/SilKitDemoLin Demos/Lin/SilKitConfig_DemoLin.json LinMaster

# LIN Slave:
build/Release/SilKitDemoLin Demos/Lin/SilKitConfig_DemoLin.json LinSlave

# System Controller:
build/Release/sil-kit-system-controller LinSlave LinMaster
Notes
- The LIN demo can only run in a synchronized mode.
- Both Master and Slave sleep for 500 milliseconds per simulation step to slow down execution.

FlexRay Demo¶

Abstract FlexRay Demo for a FlexRay cluster containing two nodes
Source location Demos/FlexRay
Requirements
  • sil-kit-system-controller
  • Network simulator (mandatory)
  • sil-kit-system-monitor (optional)
Parameters
<ParticipantConfiguration.json|yaml>
File name of the ParticipantConfiguraiton to be used; use SilKitConfig_DemoFlexRay.json for an example configuration.
<ParticipantName>
The name of the participant within the simulation; must either be Node0 or Node1.
[RegistryUri]
The silkit:// URI of the registry to connect to; defaults to silkit://localhost:8500 (optional).
Parameter Example 
# Creates a FlexRay Process for Node 0 with the default registry URI:
build/Release/SilKitDemoFlexray Demos/FlexRay/SilKitConfig_DemoFlexray.json Node0
System Example 
# Network simulator (assumed to be in PATH, necessary):
NetworkSimulator Demos/FlexRay/NetworkSimulatorConfig.json

# System Monitor (optional):
build/Release/sil-kit-system-monitor

# Node 0:
build/Release/SilKitDemoFlexray Demos/FlexRay/SilKitConfig_DemoFlexray.json Node0

# Node 1:
build/Release/SilKitDemoFlexray Demos/FlexRay/SilKitConfig_DemoFlexray.json Node1

# System Controller:
build/Release/sil-kit-system-controller Node0 Node1 NetworkSimulator
Notes Starting the FlexRay cycle takes quite some time, which is accurately modeled by the NetworkSimulator. It takes somewhat between 50 and 100 ms until the first FlexRay messages are transmitted.

Publish & Subscribe Demo¶

Abstract Data Message Publish Subscribe Demo for a set of Publishers/Subscribers
Source location Demos/PubSub
Requirements
  • sil-kit-system-controller
  • sil-kit-system-monitor (optional)
Parameters
<ParticipantConfiguration.json|yaml>
File name of the ParticipantConfiguraiton to be used; use SilKitConfig_DemoPubSub.json for an example configuration.
<ParticipantName>
The name of the participant within the simulation; must either be PubSub1, PubSub2, Subscriber1 or Subscriber2.
[RegistryUri]
The silkit:// URI of the registry to connect to; defaults to silkit://localhost:8500 (optional).
Parameter Example 
# Creates a combined publisher and subscriber with the default registry URI:
build/Release/SilKitDemoPubSub Demos/PubSub/SilKitConfig_DemoPubSub.json PubSub1
System Example 
# System Monitor (optional):
build/Release/sil-kit-system-monitor

# Publisher 1:
build/Release/SilKitDemoPubSub Demos/PubSub/SilKitConfig_DemoPubSub.json PubSub1

# Publisher 2:
build/Release/SilKitDemoPubSub Demos/PubSub/SilKitConfig_DemoPubSub.json PubSub2

# Subscriber 1:
build/Release/SilKitDemoPubSub Demos/PubSub/SilKitConfig_DemoPubSub.json Subscriber1

# Subscriber 2:
build/Release/SilKitDemoPubSub Demos/PubSub/SilKitConfig_DemoPubSub.json Subscriber2

# System Controller:
build/Release/sil-kit-system-controller PubSub1 PubSub2 Subscriber1 Subscriber2
Notes Any combination of publishers or subscribers is applicable for this demo.
